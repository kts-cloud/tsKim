Section 03

Part B: 암호화 아카이브 상세

PART B

AES-256 패키징 + Runtime Extract + Cleanup

B-1

패키징 스크립트 (빌드 타임)

rte_*-26.dll (55개, rte_argparse 포함) + hwio.dll ↓ System.IO.Compression.ZipFile 압축 ↓ System.Security.Cryptography.Aes (AES-256-CBC) 암호화 ↓ drivers.dat (단일 암호화 아카이브, ~2.3MB (압축))

새 파일: DPDK/01_Windows/scripts/pack_drivers.ps1

  • ZIP 압축 → AES-256-CBC 암호화 (IV = 랜덤 16바이트, 파일 선두에 저장)
  • 키: 앱 내 난독화 하드코딩 또는 환경변수 DPDK_HW_KEY
  • 출력: drivers.dat — 확장자로 내용 추론 불가
B-2

DpdkManager — PreloadDrivers 전면 수정

private void PreloadDrivers() { string baseDir = AppDomain.CurrentDomain.BaseDirectory; string archivePath = Path.Combine(baseDir, "drivers.dat"); // 이전 세션 잔여 temp 폴더 정리 CleanupStaleDriverDirs(); // 랜덤 temp 폴더에 추출 _driverTempDir = Path.Combine( Path.GetTempPath(), $"hw_{Guid.NewGuid():N}"); Directory.CreateDirectory(_driverTempDir); ExtractEncryptedArchive(archivePath, _driverTempDir); // Windows DLL 검색 경로에 temp 폴더 추가 SetDllDirectoryW(_driverTempDir); Environment.SetEnvironmentVariable("PATH", _driverTempDir + ";" + Environment.GetEnvironmentVariable("PATH")); // CoreLibs만 명시적 로드 (23개, 순서 중요) // 나머지는 디렉토리에 존재 → Windows 의존성 해석 시 자동 로드 // (전체 LoadLibraryA하면 tailq 이중 등록 PANIC) foreach (var lib in CoreLibs) { string path = Path.Combine(_driverTempDir, lib); if (File.Exists(path)) LoadLibraryA(path); } // hwio.dll 명시적 로드 (P/Invoke 전에) LoadLibraryA(Path.Combine(_driverTempDir, "hwio.dll")); }
B-3

복호화 메서드

private static void ExtractEncryptedArchive(string archivePath, string outputDir) { byte[] key = DeriveKey(); // 난독화된 키 byte[] encrypted = File.ReadAllBytes(archivePath); using var aes = Aes.Create(); aes.Key = key; aes.IV = encrypted[..16]; // IV = 파일 첫 16바이트 using var decryptor = aes.CreateDecryptor(); byte[] zipData = decryptor.TransformFinalBlock( encrypted, 16, encrypted.Length - 16); using var zipStream = new MemoryStream(zipData); using var archive = new ZipArchive(zipStream, ZipArchiveMode.Read); archive.ExtractToDirectory(outputDir); }
B-4

앱 종료 시 Cleanup

public void Cleanup() { // 포트 정지 + EAL cleanup foreach (var port in _ports.Where(p => p.IsSetup)) { hw_eth_dev_stop(port.PortId); hw_eth_dev_close(port.PortId); } hw_eal_cleanup(); // DLL 핸들 해제 + SetDllDirectory 초기화 foreach (var handle in _loadedHandles) FreeLibrary(handle); SetDllDirectoryW(null); // temp 폴더 삭제 if (_driverTempDir != null && Directory.Exists(_driverTempDir)) { try { Directory.Delete(_driverTempDir, true); } catch { /* 종료 중 — 무시 */ } } }

비정상 종료 대비: 앱 시작 시 CleanupStaleDriverDirs()가 이전 세션의 hw_* 잔여 폴더 정리.

Section 07

현장 적용 중 발견된 이슈

1

rte_argparse-26.dll 누락 → DllNotFoundException

근본 원인: drivers.dat에 55개 DLL 중 rte_argparse-26.dll이 CoreLibs 화이트리스트에서 빠져 있어 의존성 해석 실패.

해결: CoreLibs 화이트리스트에 rte_argparse-26.dll 추가하여 해결.

2

tailq 이중 등록 PANIC

근본 원인: Directory.GetFiles()로 전체 DLL을 LoadLibraryA 시 DPDK 내부 tailq 자료구조가 이중 등록되어 PANIC 발생.

해결: CoreLibs만 명시적 로드, 나머지는 디렉토리에 존재시키고 Windows 의존성 해석에 맡김.

3

.def NONAME 적용 (해결)

근본 원인: 현장 PC build_shim.ps1에 .def 파일 경로가 미반영. hwio.dll 재빌드 필요.

우회: P/Invoke를 ordinal 기반에서 이름 기반 (hw_*)으로 변경하여 정상 동작.

4

앱 폴더 추출 시 DLL 삭제 불가

근본 원인: 프로세스가 DLL을 로드한 상태에서 앱 폴더 내 DLL 파일 삭제 불가.

해결: temp 폴더 방식으로 전환. 종료 시 FreeLibrary 후 temp 폴더 삭제.
