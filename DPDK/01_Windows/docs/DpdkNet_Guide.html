개요

DpdkNet은 Windows 환경의 DPDK(Data Plane Development Kit)를 C# 애플리케이션에서 쉽게 사용할 수 있도록 만든 .NET 클래스 라이브러리입니다.

DPDK의 고성능 패킷 처리 기능을 P/Invoke(C Shim DLL)를 통해 캡슐화하여, 별도의 네이티브 코딩 없이 C# 프로젝트 참조만으로 초고속 UDP 송수신, Echo 서버, Request/Response 패턴 등을 구현할 수 있습니다.

주요 특징
  • 프로젝트 참조 하나로 DPDK 기능 사용 (DpdkNet.dll)
  • 싱글톤 패턴의 DpdkManager로 간편한 초기화/정리
  • 4가지 엔진: TX, RX, Echo, Request/Response
  • IDpdkContext 인터페이스를 통한 의존성 주입 (DI) 지원
  • 스레드 안전한 ConcurrentQueue 기반 패킷 전달
  • 실시간 성능 통계 (PPS, Mbps, RTT)

    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
  </PropertyGroup>

  <ItemGroup>
    <ProjectReference Include="..\DpdkNet\DpdkNet.csproj" />
  </ItemGroup>

DPDK 초기화

모든 DPDK 작업의 시작점입니다. DpdkManager.Instance 싱글톤을 통해 EAL(Environment Abstraction Layer)을 초기화하고 NIC 포트를 설정합니다.

기본 초기화

using DpdkNet;

// 상태 변경 이벤트 구독 (초기화 진행 로그)
DpdkManager.Instance.StatusChanged += (sender, e) =>
{
    Console.WriteLine($"[{e.State}] {e.Message}");
};

// 기본 옵션으로 초기화
await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

// 상태 확인
if (DpdkManager.Instance.State == DpdkState.Ready)
{
    Console.WriteLine("DPDK 초기화 성공!");
    Console.WriteLine($"DPDK 버전: {DpdkManager.Instance.DpdkVersion}");
    Console.WriteLine($"사용 포트: {DpdkManager.Instance.PortId}");
}

상세 옵션 초기화

var options = new DpdkInitOptions
{
    CoreMask   = "0-3",      // CPU 코어 0~3 사용
    MemoryMb   = 1024,       // hugepage 메모리 1GB 할당
    LogLevel   = "*:warning", // 로그 레벨 (error, warning, info, debug)
    PortId     = 0,          // 사용할 이더넷 포트 번호
    MbufPoolSize = 16383,    // 패킷 버퍼 풀 크기 (2^n - 1 권장)
    LinkSpeeds = 0           // 0 = 자동 협상 (auto-negotiation)
};

await DpdkManager.Instance.InitializeAsync(options);

상태 모니터링

StatusChanged 이벤트로 초기화 진행 상황과 오류를 실시간으로 모니터링합니다.

DpdkManager.Instance.StatusChanged += (sender, e) =>
{
    switch (e.State)
    {
        case DpdkState.Initializing:
            Console.ForegroundColor = ConsoleColor.Yellow;
            Console.WriteLine($"  ⏳ {e.Message}");
            break;
        case DpdkState.Ready:
            Console.ForegroundColor = ConsoleColor.Green;
            Console.WriteLine($"  ✓ {e.Message}");
            break;
        case DpdkState.Error:
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine($"  ✗ {e.Message}");
            break;
    }
    Console.ResetColor();
};

종료 및 정리

중요: 반드시 Cleanup 호출 DPDK는 hugepage 메모리를 사용합니다. Cleanup() 또는 Dispose()를 호출하지 않으면 메모리가 해제되지 않아 시스템 재시작이 필요할 수 있습니다. 
// 방법 1: using 패턴 (권장)
using var dpdk = DpdkManager.Instance;
await dpdk.InitializeAsync(new DpdkInitOptions());
// ... 작업 수행 ...
// using 블록 종료 시 자동 Dispose() → Cleanup()

// 방법 2: 명시적 Cleanup
try
{
    await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());
    // ... 작업 수행 ...
}
finally
{
    DpdkManager.Instance.Cleanup();
}

// 방법 3: 프로세스 종료 시 안전망
AppDomain.CurrentDomain.ProcessExit += (s, e) =>
{
    try { DpdkManager.Instance.Cleanup(); } catch { }
};

UDP TX (송신) 엔진

고속 UDP 패킷 송신 엔진입니다. 목적지 MAC/IP/포트를 지정하고, PPS(Packets Per Second) 속도 제한을 설정할 수 있습니다.

using DpdkNet;
using DpdkNet.Config;
using DpdkNet.Engine;
using DpdkNet.Stats;
using DpdkNet.Utilities;

// 1. DPDK 초기화
await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions { PortId = 0 });

// 2. 성능 카운터 생성 및 시작
var perf = new PerformanceCounter();
perf.Start();

// 3. TX 엔진 생성 (IDpdkContext 주입)
using var tx = new DpdkUdpTxEngine(DpdkManager.Instance, perf);

// 4. 송신 설정
var config = new TxConfig
{
    DstMac      = NetUtils.ParseMac("AA:BB:CC:DD:EE:FF"),
    DstIp       = "192.168.1.100",
    DstPort     = 5000,
    SrcIp       = "192.168.1.10",
    SrcPort     = 4000,
    PayloadSize = 64,        // 페이로드 바이트 수
    TargetPps   = 100000,    // 초당 10만 패킷 (0 = 무제한)
    PayloadText = "Hello DPDK!"  // null이면 0x00 패딩
};

// 5. 송신 시작
tx.Start(config);
Console.WriteLine("TX 엔진 시작. 5초 후 중지...");

// 6. 실시간 통계 모니터링
for (int i = 0; i < 5; i++)
{
    await Task.Delay(1000);
    var snap = perf.TakeSnapshot();
    Console.WriteLine($"TX: {snap.TxPackets:N0} pkts, " +
                      $"{snap.TxPps:N0} PPS, {snap.TxMbps:F1} Mbps");
}

// 7. 중지
tx.Stop();

// 최종 결과
var final = perf.TakeSnapshot();
Console.WriteLine($"\n총 송신: {final.TxPackets:N0} 패킷, {final.TxBytes:N0} 바이트");
Console.WriteLine($"Peak: {perf.PeakTxPps:N0} PPS, {perf.PeakTxMbps:F1} Mbps");

응답 수신 (TX + RX) TX 엔진은 ResponseQueue 속성을 통해 응답 패킷도 수신할 수 있습니다. TX 스레드 내에서 RX polling을 동시에 수행합니다. 
// TX 중 수신된 응답 확인
while (tx.ResponseQueue.TryDequeue(out var resp))
{
    Console.WriteLine($"응답: {resp.SrcIp}:{resp.SrcPort} → {resp.PayloadText}");
}

UDP RX (수신) 엔진

UDP 패킷 수신 전용 엔진입니다. 포트 번호와 IP 주소로 필터링이 가능하며, 수신된 패킷은 ConcurrentQueue로 전달됩니다.

using DpdkNet;
using DpdkNet.Engine;
using DpdkNet.Stats;

await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

var perf = new PerformanceCounter();
perf.Start();

// RX 엔진 생성 + 필터 설정
using var rx = new DpdkUdpRxEngine(DpdkManager.Instance, perf)
{
    FilterPort   = 5000,          // 이 포트로 오는 패킷만 수신 (0=전체)
    FilterIp     = "192.168.1.10", // 이 IP로 오는 패킷만 (null=전체)
    MaxQueueSize = 10000          // 큐 최대 크기 (초과 시 드롭)
};

// 수신 시작
rx.Start();
Console.WriteLine("RX 엔진 시작. 패킷 대기 중...");

// 패킷 처리 루프
var cts = new CancellationTokenSource(TimeSpan.FromSeconds(10));

while (!cts.Token.IsCancellationRequested)
{
    while (rx.PacketQueue.TryDequeue(out var pkt))
    {
        Console.WriteLine(
            $"[{pkt.Timestamp:HH:mm:ss.fff}] " +
            $"{pkt.SrcIp}:{pkt.SrcPort} → {pkt.DstIp}:{pkt.DstPort} " +
            $"({pkt.DataLen}B) {pkt.PayloadText}");
    }
    await Task.Delay(10);
}

rx.Stop();

var snap = perf.TakeSnapshot();
Console.WriteLine($"총 수신: {snap.RxPackets:N0} 패킷");

UDP Echo (반향) 엔진

수신된 UDP 패킷의 MAC/IP/포트를 뒤집어 즉시 반송하는 Echo 서버입니다. 네트워크 연결 테스트와 지연 시간 측정에 유용합니다.

using DpdkNet;
using DpdkNet.Engine;
using DpdkNet.Stats;

await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

var perf = new PerformanceCounter();
perf.Start();

// Echo 엔진 시작 — 모든 수신 UDP를 즉시 반송
using var echo = new DpdkEchoEngine(DpdkManager.Instance, perf);
echo.Start();

Console.WriteLine("Echo 서버 실행 중. Ctrl+C로 종료...");
Console.CancelKeyPress += (s, e) => { e.Cancel = true; echo.Stop(); };

// 통계 모니터링
while (echo.IsRunning)
{
    await Task.Delay(2000);
    var snap = perf.TakeSnapshot();
    Console.WriteLine(
        $"Echo: RX {snap.RxPps:N0} PPS → TX {snap.TxPps:N0} PPS");
}
Echo 동작 원리 Echo 엔진은 수신 패킷의 Ethernet 헤더(src/dst MAC), IPv4 헤더(src/dst IP), UDP 헤더(src/dst Port)를 스왑한 후 IP 체크섬을 재계산하여 송신합니다. 페이로드는 변경 없이 그대로 반송됩니다.

Request/Response 엔진

지정된 횟수만큼 UDP 요청을 보내고 응답을 기다리며, RTT(Round-Trip Time)를 측정하는 엔진입니다. ARP 자동 해석, 파이프라인(윈도우) 모드, NIC 워밍업 등 고급 기능을 제공합니다.

기본 사용법 (동기 모드)

using DpdkNet;
using DpdkNet.Config;
using DpdkNet.Engine;
using DpdkNet.Stats;
using DpdkNet.Utilities;

await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

var perf = new PerformanceCounter();
perf.Start();

using var rr = new DpdkUdpReqRespEngine(DpdkManager.Instance, perf);

var config = new ReqRespConfig
{
    DstMac      = NetUtils.ParseMac("AA:BB:CC:DD:EE:FF"),
    DstIp       = "192.168.1.100",
    DstPort     = 5000,
    SrcIp       = "192.168.1.10",
    SrcPort     = 4000,
    PayloadSize = 64,
    TimeoutMs   = 1000,     // 응답 대기 시간 (ms)
    RepeatCount = 100,      // 요청 횟수 (0 = 무한)
    WindowSize  = 1,        // 1 = 동기 모드 (한 번에 하나씩)
    EnableWarmup = true     // NIC/DMA 워밍업 수행
};

rr.Start(config);

// 완료 대기 + 실시간 결과 처리
while (rr.IsRunning || !rr.ResultQueue.IsEmpty)
{
    while (rr.ResultQueue.TryDequeue(out var result))
    {
        if (result.Success)
            Console.WriteLine($"  #{result.SeqNumber}: RTT={result.RttMs:F3}ms");
        else
            Console.WriteLine($"  #{result.SeqNumber}: TIMEOUT");
    }
    await Task.Delay(50);
}

// RTT 통계 출력
var stats = rr.Stats;
Console.WriteLine($"\n── RTT 통계 ──");
Console.WriteLine($"  전송: {stats.Sent}  수신: {stats.Received}  타임아웃: {stats.Timeouts}");
Console.WriteLine($"  Min: {stats.MinRtt:F3}ms  Avg: {stats.AvgRtt:F3}ms  Max: {stats.MaxRtt:F3}ms");

파이프라인 모드 (고성능)

WindowSize > 1로 설정하면 파이프라인 모드가 활성화됩니다. 여러 요청을 동시에 보내고 응답을 병렬로 수집하여 처리량을 극대화합니다.

var pipelineConfig = new ReqRespConfig
{
    DstMac      = NetUtils.ParseMac("AA:BB:CC:DD:EE:FF"),
    DstIp       = "192.168.1.100",
    DstPort     = 5000,
    SrcIp       = "192.168.1.10",
    SrcPort     = 4000,
    PayloadSize = 64,
    TimeoutMs   = 500,
    RepeatCount = 10000,    // 1만 회 요청
    WindowSize  = 32,       // 동시 32개 요청 (파이프라인)
    EnableWarmup = true
};

ReqRespConfig 속성

속성타입기본값설명

Multi-Channel Request/Response (shim_reqresp_once_mc)

여러 채널(CH1~CH4)이 동일 NIC에서 동시에 req/resp를 수행할 때, 패킷 충돌 없이 각 채널이 자신의 응답만 정확히 수신할 수 있도록 설계된 네이티브 함수입니다.

기존 shim_reqresp_once와의 차이
  • rte_spinlock_t g_rx_lock으로 rte_eth_rx_burst 접근을 보호 (다중 스레드 안전)
  • 매칭되지 않는 패킷을 drop 하지 않고 g_spill[128] 버퍼에 enqueue
  • shim_rx_burst는 spill 버퍼를 먼저 배출한 후 NIC에서 추가 수신
  • 다른 채널의 응답 패킷이 손실되지 않음

매칭 기준

파라미터바이트 오더비교 대상
expected_src_ipNetwork byte order수신 패킷의 IPv4 Source Address
expected_dst_portHost byte order (내부에서 rte_cpu_to_be_16 변환)수신 패킷의 UDP Destination Port
주의: expected_src_ip 바이트 오더 expected_src_ip는 반드시 network byte order로 전달해야 합니다. NetUtils.IpToUint()는 host byte order를 반환하므로, 반드시 NetUtils.Htonl()로 변환하세요.

올바른 예: NetUtils.Htonl(NetUtils.IpToUint("169.254.199.11"))
잘못된 예: NetUtils.IpToUint("169.254.199.11") // host byte order → 매칭 실패!

Spill Buffer 동작 원리

// dpdk_shim.c 내부 구조
static struct rte_mbuf *g_spill[128];       // 비매칭 패킷 임시 저장
static volatile int     g_spill_count = 0;
static rte_spinlock_t    g_rx_lock;           // rx_burst 보호용 spinlock

// shim_rx_burst 호출 시:
//   1. spinlock 획득
//   2. g_spill[] 에서 먼저 배출 (FIFO)
//   3. 남은 공간에 rte_eth_rx_burst() 로 NIC 수신
//   4. spinlock 해제

// shim_reqresp_once_mc RX 루프:
//   1. spinlock 획득 → rte_eth_rx_burst
//   2. 각 패킷의 src_ip / dst_port 비교
//   3. 매칭 → 응답으로 반환
//   4. 비매칭 → g_spill[] 에 enqueue (다른 채널이 나중에 수신)
//   5. spinlock 해제

사용 예제

// PgDpdkServer.Send() 내부 — 동기 TX+RX 경로
var templatePkt = BuildUdpPacket(dstMac, dstIpNet,
    (ushort)localPort, (ushort)peerPort, payload);
var respBuf = new byte[2048];
var result = new ShimReqRespResult();

// expected_src_ip: PG의 IP를 network byte order로 전달
var expectedSrcIp = NetUtils.Htonl(NetUtils.IpToUint(peerIpStr));

int rc = DpdkInterop.shim_reqresp_once_mc(
    portId, mbufPool,
    templatePkt, (ushort)templatePkt.Length, 0,
    expectedSrcIp, (ushort)localPort,
    3000,                        // timeout 3초
    respBuf, (ushort)respBuf.Length,
    ref result, localMac, localIpNet);

if (rc == 0 && result.Status == 0)
{
    var respStr = Encoding.ASCII.GetString(respBuf, 0, result.RespLen);
    Console.WriteLine($"응답: {respStr}, RTT: {result.RttMs:F3}ms");
}

FTP Engine (lwIP 기반)

DPDK NIC을 통해 lwIP TCP/IP 스택으로 FTP 파일 전송을 수행하는 엔진입니다. OS 소켓을 사용하지 않고 DPDK의 커널 바이패스를 활용하여 고성능 파일 전송을 구현합니다.

Unified Dispatcher: UDP + FTP 동시 동작 shim_dispatch_poll() 기반 Unified Dispatcher가 RX 패킷을 자동 분류합니다:
  • UDP 패킷 → C# ProcessRxPacket으로 전달 (PG 통신 지속)
  • TCP/ARP 패킷 → lwIP 스택으로 전달 (FTP 통신)
FTP 중에도 pg.status, tcon 등 UDP 명령이 정상 동작합니다.
멀티세션 FTP를 지원하여 최대 4개 CH가 동시에 FTP를 수행할 수 있습니다 (sessionId = 0~3).

기본 사용법

using DpdkNet;
using DpdkNet.Config;
using DpdkNet.Engine;
using DpdkNet.Models;

await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

// sessionId: CH 인덱스 (0~3). 각 CH가 독립 FTP 세션 사용
using var ftp = new DpdkFtpEngine(DpdkManager.Instance, sessionId: 0);

// 1. lwIP 스택 초기화
var config = new FtpConfig
{
    ServerIp   = "192.168.0.1",
    ServerPort = 21,
    Username   = "admin",
    Password   = "password",
    LocalIp    = "192.168.0.2",
    Netmask    = "255.255.255.0",
    Gateway    = "192.168.0.1",
    TimeoutMs  = 10000
};

if (!ftp.InitLwip(config))
{
    Console.WriteLine($"lwIP 초기화 실패: {ftp.LastError}");
    return;
}

// 2. FTP 연결
bool connected = await ftp.ConnectAsync(
    config.ServerIp, config.ServerPort,
    config.Username, config.Password, config.TimeoutMs);

if (!connected)
{
    Console.WriteLine($"FTP 연결 실패: {ftp.LastError}");
    return;
}

// 3. 디렉토리 탐색
string? pwd = await ftp.PwdAsync();
Console.WriteLine($"현재 디렉토리: {pwd}");

string? listing = await ftp.ListAsync(10000);
Console.WriteLine($"파일 목록:\n{listing}");

await ftp.CwdAsync("/upload");

// 4. 파일 다운로드
byte[]? data = await ftp.DownloadAsync("test.bin", 30000);
if (data != null)
    File.WriteAllBytes("downloaded.bin", data);

// 5. 파일 업로드
byte[] uploadData = File.ReadAllBytes("local_file.bin");
bool uploaded = await ftp.UploadAsync("remote_file.bin", uploadData, 30000);

// 6. 연결 종료
await ftp.DisconnectAsync();
ftp.StopLwip();

멀티세션 FTP (4CH 동시 전송)

각 CH가 독립 sessionId를 사용하여 동시에 FTP 파일 전송을 수행할 수 있습니다. lwIP 스택은 레퍼런스 카운팅으로 관리되어 첫 세션이 init, 마지막 세션이 stop을 수행합니다.

// CH별 동시 FTP 다운로드 예제
var tasks = new List<Task>();
for (int ch = 0; ch < 4; ch++)
{
    int sessionId = ch;
    tasks.Add(Task.Run(async () =>
    {
        using var ftp = new DpdkFtpEngine(DpdkManager.Instance, sessionId);
        ftp.InitLwip(config);  // 레퍼런스 카운팅: 첫 호출만 실제 init
        try
        {
            await ftp.ConnectAsync(pgIp[sessionId], 21, user, pass, 10000);
            byte[]? data = await ftp.DownloadAsync(remotePath, 30000);
            await ftp.DisconnectAsync();
        }
        finally
        {
            ftp.StopLwip();  // 레퍼런스 카운팅: 마지막 호출만 실제 stop
        }
    }));
}
await Task.WhenAll(tasks);
네이티브 세션 풀 dpdk_shim.dll 내부에 ftp_client_t g_ftp_sessions[4] 배열이 할당되어 있습니다. 각 sessionId는 독립적인 FTP 상태머신을 사용하므로 서로 간섭하지 않습니다. _ex 접미사가 붙은 P/Invoke 함수들(shim_ftp_connect_sync_ex 등)이 세션 ID를 받습니다.

NIC Coordinator (동시 멀티세션 FTP)

Unified Dispatcher (shim_dispatch_poll) 아키텍처에서 UDP와 FTP(TCP)는 동시에 동작합니다. DpdkNicCoordinator는 레퍼런스 카운팅으로 lwIP external RX 모드를 관리합니다: 첫 FTP 세션 시작 시 lwIP external RX를 활성화하고, 마지막 세션 종료 시 비활성화합니다.

레퍼런스 카운팅 동작 흐름
  1. AcquireFtpAccessAsync() — Interlocked.Increment(_ftpActiveCount)
  2. 첫 세션 (activeCount == 1): SetLwipExternalRx(true) — Dispatcher가 TCP/ARP를 lwIP에 전달 시작
  3. FTP 작업 수행 (각 CH가 독립 sessionId로 동시 전송 가능)
  4. lease.DisposeAsync() → Interlocked.Decrement(_ftpActiveCount)
  5. 마지막 세션 (activeCount == 0): SetLwipExternalRx(false)
// DpdkNicCoordinator 사용 예제 (CommPgDriver 내부)
// PgIndex = CH 인덱스 (0~3) → sessionId로 사용
var lease = await _nicCoordinator.AcquireFtpAccessAsync();
try
{
    // sessionId = PgIndex → CH별 독립 FTP 세션
    var ftpEngine = new DpdkFtpEngine(_dpdkManager, PgIndex);
    try
    {
        var config = CreatePgFtpConfig();
        ftpEngine.InitLwip(config);  // 레퍼런스 카운팅 init

        await ftpEngine.ConnectAsync(pgIpAddress, 21,
            username, password, 10000);

        // FTP 다운로드/업로드 수행 (다른 CH도 동시에 FTP 가능)
        byte[]? data = await ftpEngine.DownloadAsync(remoteFile, 30000);

        await ftpEngine.DisconnectAsync();
    }
    finally
    {
        ftpEngine.StopLwip();  // 레퍼런스 카운팅 stop
        ftpEngine.Dispose();
    }
}
finally
{
    await lease.DisposeAsync(); // activeCount 감소, 마지막이면 external RX 해제
}
로그 확인 NicCoordinator 동작은 앱 로그에서 다음 메시지로 확인할 수 있습니다:
  • [DpdkNicCoordinator] FTP 활성화 (activeCount=N)
  • [DpdkNicCoordinator] external RX 활성화 (첫 FTP 세션)
  • [DpdkNicCoordinator] FTP 해제 (activeCount=N)
  • [DpdkNicCoordinator] external RX 비활성화 (마지막 FTP 세션 종료)

다중 엔진 운용

하나의 DpdkManager에서 여러 엔진을 동시에 실행할 수 있습니다. 각 엔진은 자체 스레드에서 동작합니다.

await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions());

var txPerf = new PerformanceCounter();
var rxPerf = new PerformanceCounter();
txPerf.Start();
rxPerf.Start();

// TX와 RX를 동시에 실행
using var tx = new DpdkUdpTxEngine(DpdkManager.Instance, txPerf);
using var rx = new DpdkUdpRxEngine(DpdkManager.Instance, rxPerf)
{
    FilterPort = 5000
};

tx.Start(new TxConfig
{
    DstMac = NetUtils.ParseMac("AA:BB:CC:DD:EE:FF"),
    DstIp = "192.168.1.100", DstPort = 5000,
    SrcIp = "192.168.1.10",  SrcPort = 4000,
    PayloadSize = 128, TargetPps = 50000
});
rx.Start();

// 5초간 모니터링
for (int i = 0; i < 5; i++)
{
    await Task.Delay(1000);
    var txSnap = txPerf.TakeSnapshot();
    var rxSnap = rxPerf.TakeSnapshot();
    Console.WriteLine(
        $"TX: {txSnap.TxPps:N0} PPS | RX: {rxSnap.RxPps:N0} PPS");
}

tx.Stop();
rx.Stop();

ARP 자동 해석

DpdkUdpReqRespEngine은 시작 시 자동으로 ARP Request를 보내 목적지 MAC 주소를 해석합니다. DstMac에 broadcast MAC(FF:FF:FF:FF:FF:FF)을 넣으면 ARP가 자동 수행됩니다.

DispatchPoll 모드에서의 ARP 처리 Unified Dispatcher (shim_dispatch_poll) 사용 시 ARP 패킷 처리:
  • lwIP 실행 중: ARP 패킷이 lwIP 스택으로 전달되어 자동 응답
  • lwIP 미실행: ARP 패킷이 C# out_udp_pkts로 반환되어 PgDpdkServer.HandleArpPacket()이 직접 ARP Reply 생성
이 이중 경로 설계로 lwIP 실행 여부와 관계없이 PG 보드가 항상 PC의 MAC 주소를 해석할 수 있습니다. 
var config = new ReqRespConfig
{
    // broadcast MAC → ARP 자동 해석
    DstMac      = new byte[] { 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF },
    DstIp       = "192.168.1.100",
    DstPort     = 5000,
    SrcIp       = "192.168.1.10",
    SrcPort     = 4000,
    PayloadSize = 64,
    RepeatCount = 100
};

// DiagQueue로 ARP 진행 상황 확인
while (rr.DiagQueue.TryDequeue(out var diag))
{
    Console.WriteLine($"[DIAG] {diag}");
    // 예: "ARP: resolved 192.168.1.100 → AA:BB:CC:DD:EE:FF"
}

베스트 프랙티스

1. 반드시 관리자 권한으로 실행 DPDK는 hugepage 메모리와 PCI 디바이스에 직접 접근합니다. 관리자 권한 없이는 초기화가 실패합니다.
2. Cleanup 보장 using 패턴, try-finally, 또는 ProcessExit 이벤트를 활용하여 hugepage 메모리를 반드시 해제하세요.
3. MbufPoolSize 설정 2^n - 1 값을 사용하세요 (예: 8191, 16383, 32767). DPDK 내부 최적화와 관련됩니다.
4. 엔진 생명주기 엔진을 Stop하기 전에 반드시 IsRunning을 확인하세요. 이미 중지된 엔진의 Stop은 안전하지만, 불필요한 호출입니다.
5. PerformanceCounter 공유 하나의 PerformanceCounter를 여러 엔진에서 공유하면 TX/RX 합산 통계를 얻을 수 있습니다. 개별 통계가 필요하면 엔진마다 별도로 생성하세요.
6. 큐 오버플로우 방지 RX 엔진의 PacketQueue를 주기적으로 소비하지 않으면 메모리가 증가합니다. MaxQueueSize를 적절히 설정하세요.

FTP 재연결 패턴

FTP 작업은 서버 타임아웃 (421 응답), 네트워크 장애, 또는 유휴 연결 해제 등의 이유로 실패할 수 있습니다. ReconnectAsync를 활용하여 자동 재연결 + 재시도 패턴을 구현할 수 있습니다.

// 작업 실패 시 자동 재연결 + 재시도 패턴
async Task<string?> SafeListAsync(DpdkFtpEngine ftp, FtpConfig config)
{
    string? result = await ftp.ListAsync(config.TimeoutMs);
    if (result == null)
    {
        // 재연결 시도
        bool reconnected = await ftp.ReconnectAsync(config);
        if (reconnected)
            result = await ftp.ListAsync(config.TimeoutMs);
    }
    return result;
}
버퍼 크기 제한
  • 다운로드: 최대 64MB
  • LIST 응답: 최대 64KB
  • PWD 응답: 최대 512B

전체 예제: 콘솔 애플리케이션

DPDK를 초기화하고 UDP 패킷을 송신한 후 통계를 출력하는 완전한 콘솔 애플리케이션입니다.

using DpdkNet;
using DpdkNet.Config;
using DpdkNet.Engine;
using DpdkNet.Stats;
using DpdkNet.Utilities;

namespace MyDpdkApp;

class Program
{
    static async Task Main(string[] args)
    {
        // ── 안전망: 비정상 종료 시 메모리 해제 ──
        AppDomain.CurrentDomain.ProcessExit += (s, e) =>
        {
            try { DpdkManager.Instance.Cleanup(); } catch { }
        };

        // ── 1. 초기화 ──
        Console.WriteLine("DPDK 초기화 중...");

        DpdkManager.Instance.StatusChanged += (s, e) =>
            Console.WriteLine($"  [{e.State}] {e.Message}");

        await DpdkManager.Instance.InitializeAsync(new DpdkInitOptions
        {
            CoreMask     = "0",
            MemoryMb     = 512,
            LogLevel     = "*:error",
            PortId       = 0,
            MbufPoolSize = 8191
        });

        if (DpdkManager.Instance.State != DpdkState.Ready)
        {
            Console.WriteLine($"초기화 실패: {DpdkManager.Instance.ErrorMessage}");
            return;
        }

        Console.WriteLine($"✓ 초기화 완료 (v{DpdkManager.Instance.DpdkVersion})");
        Console.WriteLine($"  MAC: {NetUtils.FormatMac(DpdkManager.Instance.LocalMac)}");

        // ── 2. 링크 상태 확인 ──
        var link = DpdkManager.Instance.GetLinkInfo();
        Console.WriteLine($"  Link: {(link.link_status == 1 ? "UP" : "DOWN")} @ {link.link_speed} Mbps");

        // ── 3. TX 엔진 실행 ──
        var perf = new PerformanceCounter();
        perf.Start();

        using var tx = new DpdkUdpTxEngine(DpdkManager.Instance, perf);

        tx.Start(new TxConfig
        {
            DstMac      = NetUtils.ParseMac("AA:BB:CC:DD:EE:FF"),
            DstIp       = "192.168.1.100",
            DstPort     = 5000,
            SrcIp       = "192.168.1.10",
            SrcPort     = 4000,
            PayloadSize = 64,
            TargetPps   = 100000,
            PayloadText = "Hello from DpdkNet!"
        });

        Console.WriteLine("\nTX 엔진 시작! 10초간 송신...");

        // ── 4. 실시간 통계 출력 ──
        for (int i = 0; i < 10; i++)
        {
            await Task.Delay(1000);
            var snap = perf.TakeSnapshot();
            Console.WriteLine(
                $"  [{i + 1:D2}s] TX: {snap.TxPackets:N0} pkts | " +
                $"{snap.TxPps:N0} PPS | {snap.TxMbps:F1} Mbps");
        }

        // ── 5. 중지 ──
        tx.Stop();

        // ── 6. 최종 결과 ──
        var final_ = perf.TakeSnapshot();
        var hwStats = DpdkManager.Instance.GetPortStats();

        Console.WriteLine($"\n── 최종 결과 ──");
        Console.WriteLine($"  SW 통계:");
        Console.WriteLine($"    총 송신: {final_.TxPackets:N0} pkts ({final_.TxBytes:N0} bytes)");
        Console.WriteLine($"    Peak:   {perf.PeakTxPps:N0} PPS ({perf.PeakTxMbps:F1} Mbps)");
        Console.WriteLine($"  HW 통계:");
        Console.WriteLine($"    NIC TX: {hwStats.opackets:N0} pkts");
        Console.WriteLine($"    Errors: {hwStats.oerrors:N0}");

        // ── 7. 정리 ──
        DpdkManager.Instance.Cleanup();
        Console.WriteLine("\n정리 완료.");
    }
}

  • StatusChanged 이벤트를 구독하면 초기화 단계별 진행 상황을 파악할 수 있습니다
